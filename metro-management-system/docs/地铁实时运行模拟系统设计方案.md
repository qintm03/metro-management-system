# 地铁实时运行模拟系统设计方案

## 概述

基于高德地图 API + Vue3 + SpringBoot 的地铁列车实时运行模拟系统。采用简化架构，后端提供时刻表数据，前端自主计算列车位置并使用 `moveAlong` 实现动画效果。

---

## 核心设计思想

### 为什么选择前端计算？

| 方案 | 优点 | 缺点 |
|------|------|------|
| **前端计算**（本方案） | 简单、无状态、易扩展 | 多客户端不同步 |
| **WebSocket 实时推送** | 完全同步、可统一控制 | 复杂、需要维护连接 |

**本系统采用前端计算方案**，适合演示场景，实现简单快速。

## 前端核心算法

### 1. 列车位置计算算法

```javascript
/**
 * 计算列车当前位置
 * @param {Object} train - 列车信息
 * @param {Number} currentSeconds - 当前时间（从06:00开始的秒数）
 * @returns {Object} 位置信息
 */
function calculateTrainPosition(train, currentSeconds) {
  const { startOffset, segmentTime, dwellTime, stationCount } = lineData.value
  const trainStartTime = startOffset
  
  // 还没发车
  if (currentSeconds < trainStartTime) {
    return { 
      status: 'waiting', 
      stationIndex: 0, 
      progress: 0 
    }
  }
  
  // 已过时间（从该列车发车开始算）
  const elapsed = currentSeconds - trainStartTime
  
  // 计算当前在哪一站或哪一段
  let accumulatedTime = 0
  const segmentCount = stationCount - 1
  
  for (let i = 0; i < segmentCount; i++) {
    // 停站时间
    if (elapsed >= accumulatedTime && elapsed < accumulatedTime + dwellTime) {
      return { 
        status: 'stopped', 
        stationIndex: i,
        progress: 0 
      }
    }
    accumulatedTime += dwellTime
    
    // 运行时间（使用统一的 segmentTime）
    if (elapsed >= accumulatedTime && elapsed < accumulatedTime + segmentTime) {
      const progress = (elapsed - accumulatedTime) / segmentTime
      return {
        status: 'running',
        fromIndex: i,
        toIndex: i + 1,
        progress: progress,
        remainTime: segmentTime - (elapsed - accumulatedTime)
      }
    }
    accumulatedTime += segmentTime
  }
  
  // 已到达终点
  return { 
    status: 'finished', 
    stationIndex: segmentCount 
  }
}
```

### 2. 位置更新流程

```
每秒执行一次：
  1. 获取当前模拟时间（秒）
  2. 遍历所有列车：
     a. 调用 calculateTrainPosition 计算位置
     b. 如果状态/位置变化：
        - running: 使用 moveAlong 移动到下一站
        - stopped: 停止动画，设置到站点位置
        - waiting: 停止动画，设置在起点
  3. 更新界面显示
```

### 3. moveAlong 动画执行

```javascript
function updateTrainMarker(train, position) {
  const marker = train.marker
  const stations = lineData.value.stations
  
  switch (position.status) {
    case 'running': {
      const from = stations[position.fromIndex]
      const to = stations[position.toIndex]
      
      // 计算当前经纬度
      const currentLng = from.lng + (to.lng - from.lng) * position.progress
      const currentLat = from.lat + (to.lat - from.lat) * position.progress
      
      // 使用 moveAlong 平滑移动到下一站
      marker.moveAlong(
        [[currentLng, currentLat], [to.lng, to.lat]],
        {
          duration: position.remainTime * 1000,  // 转换为毫秒
          autoRotation: true,                     // 自动旋转
          circlable: false                        // 不循环
        }
      )
      break
    }
    
    case 'stopped': {
      marker.stopMove()
      const station = stations[position.stationIndex]
      marker.setPosition([station.lng, station.lat])
      break
    }
    
    case 'waiting': {
      marker.stopMove()
      const station = stations[0]
      marker.setPosition([station.lng, station.lat])
      break
    }
  }
}
```

## 模拟参数配置

### 默认参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 首班车时间 | 06:00 | 每天开始运行时间 |
| 发车间隔 | 300秒（5分钟） | 相邻两列车的发车时间差 |
| 站间运行时间 | 180秒（3分钟） | 两站之间的运行时间（所有路段相同） |
| 停站时间 | 30秒 | 每站停靠时间 |
| 列车数量 | 5辆 | 同时运行的列车数 |

### 时间计算示例

以 1 号线（16个站点）为例：

| 列车 | 发车时间 | 到达终点时间 | 总耗时 |
|------|----------|--------------|--------|
| T001 | 06:00 | 06:52:30 | 52分30秒 |
| T002 | 06:05 | 06:57:30 | - |
| T003 | 06:10 | 07:02:30 | - |
| T004 | 06:15 | 07:07:30 | - |
| T005 | 06:20 | 07:12:30 | - |

**计算公式**：
```
总耗时 = (站点数 - 1) × (站间时间 + 停站时间) - 停站时间
      = 15 × (180 + 30) - 30
      = 15 × 210 - 30
      = 3150 - 30
      = 3120秒 = 52分钟
```

### Controller 实现

```java
@RestController
@RequestMapping("/api/metro")
public class MetroController {
    
    @Autowired
    private MetroService metroService;
    
    @GetMapping("/line/{lineId}/schedule")
    public Result getLineSchedule(@PathVariable String lineId) {
        return Result.success(metroService.getSchedule(lineId));
```

### Service 实现

```java
@Service
public class MetroService {
    
    @Autowired
    private MetroLineMapper lineMapper;
    
    @Autowired
    private MetroStationMapper stationMapper;
    
    public Map<String, Object> getSchedule(String lineId) {
        MetroLine line = lineMapper.selectByLineId(lineId);
        List<MetroStation> stations = stationMapper.selectByLineIdOrderBySequence(lineId);
        
        Map<String, Object> result = new HashMap<>();
        result.put("lineId", line.getLineId());
        result.put("lineName", line.getLineName());
        result.put("color", line.getColor());
        result.put("firstTrainTime", line.getFirstTrainTime().toString());
        result.put("lastTrainTime", line.getLastTrainTime().toString());
        result.put("interval", line.getInterval());
        result.put("segmentTime", line.getSegmentTime());
        result.put("dwellTime", line.getDwellTime());
        result.put("stations", stations.stream().map(s -> {
            Map<String, Object> station = new HashMap<>();
            station.put("id", s.getId());
            station.put("name", s.getStationName());
            station.put("lng", s.getLongitude());
            station.put("lat", s.getLatitude());
            station.put("sequence", s.getSequence());
            return station;
        }).collect(Collectors.toList()));
        
        return result;
    }
    
    public Map<String, Object> getTrains(String lineId) {
        MetroLine line = lineMapper.selectByLineId(lineId);
        int interval = line.getInterval();
        
        List<Map<String, Object>> trains = new ArrayList<>();
        for (int i = 0; i < 5; i++) {
            Map<String, Object> train = new HashMap<>();
            train.put("trainId", "T" + String.format("%03d", i + 1));
            train.put("startOffset", i * interval);
            trains.add(train);
        }
        
        Map<String, Object> result = new HashMap<>();
        result.put("trains", trains);
        return result;
    }
}
```

## 优缺点分析

### 优点

1. **实现简单**：后端只提供静态数据，无需维护实时状态
2. **无状态服务**：易于水平扩展，支持高并发
3. **前端自治**：刷新页面后能立即恢复当前状态
4. **易于调试**：时刻表数据可缓存，方便开发和测试
5. **节省资源**：无需 WebSocket 长连接，减少服务器负担

### 缺点

1. **不同步**：多个用户看到的时间可能有几秒差异
2. **无法干预**：后端无法实时控制列车（如紧急停车）
3. **安全性**：前端计算可能被篡改（演示场景可接受）

### 适用场景

✅ **适合**：
- 演示系统
- 单用户场景
- 无需精确同步的场景
- 快速原型开发

❌ **不适合**：
- 多用户需要严格同步
- 需要实时调度控制
- 高安全性要求的场景
