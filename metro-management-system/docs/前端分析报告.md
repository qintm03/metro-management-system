# 前端项目分析报告：城市智慧地铁管理系统

## 一、项目概览

这是一个**毕业设计项目**，定位为"广州地铁智慧管理平台"，采用前后端分离架构。前端技术栈为 **Vue 3 + Vite 7 + Element Plus 2**，使用高德地图 API 实现地铁线路可视化、路线规划、站点周边服务搜索和列车实时模拟等功能。

## 五、已实现功能详解

### 1. 登录系统 ([Login.vue](../src/components/Login.vue))
- 精美的渐变背景 + 卡片式登录表单
- 用户名/密码/角色三要素登录
- 默认填充测试账号 `qtm/123456`
- 登录后按角色分流：管理员 → `/manager`，普通用户 → `/user`
- **问题**：登录成功后直接将用户信息存到 `localStorage`，未使用 `auth.js` 工具函数

### 2. 运营监控大屏 ([AdminHomePage.vue](../src/components/admin/AdminHomePage.vue))
这是项目的**核心页面**，实现了：
- 高德地图加载与地铁线路可视化（折线 + 站点标记）
- 左侧面板：运营概况（线路数/站点数/换乘站数）、线路列表（可筛选）、站点搜索
- 右侧面板：站点详情、换乘信息、线路站点列表
- 广州地铁 19 条线路的颜色映射
- 响应式布局（3 栏 grid → 单栏）

### 3. 出行服务 ([TripService.vue](../src/components/user/TripService.vue))
- 左侧：起点/终点选择（线路 → 站点级联选择）
- 起终点交换按钮
- 右侧：高德地图 + Transfer 公交换乘路线规划
- **问题**：`metroLines` 数据是**硬编码**在前端的（300+ 行），而非从后端 API 获取

### 4. 站点周边服务 ([StationServices.vue](../src/components/user/StationServices.vue))
- 线路 → 站点级联选择
- 高德 PlaceSearch 搜索周边 POI
- 分类筛选：全部/餐饮/购物/生活/娱乐
- 搜索范围：500m / 1km / 2km
- POI 详情 + 步行导航

### 5. 实时轨迹模拟 ([RealTimeTrack.vue](../src/components/user/RealTimeTrack.vue))
- 从后端 API 获取线路时刻表和列车列表
- **自己计算**列车位置（基于发车偏移量 + 站间行驶时间 + 停站时间）
- 列车状态：waiting / running / stopped / finished
- 每秒更新所有列车位置
- 侧边面板：列车实时状态列表 + 站点状态（列车停靠/前往中/已通过）

### 6. 用户管理 ([UserManagement.vue](../src/components/admin/UserManagement.vue))
- 完整的 CRUD 表格
- 搜索、分页功能
- 新增/编辑弹窗表单 + 角色选择

## 七、代码质量评估

### 优点

1. **CSS 设计系统完善** — `variables.css` 定义了完整的设计 Token（颜色/字体/间距/阴影/动画），覆盖了 Element Plus 变量
2. **登录页 UI 精美** — 渐变背景 + 动画 + 毛玻璃卡片，视觉效果专业
3. **地图集成深度好** — 高德 API 使用较全面（Marker/Polyline/PlaceSearch/Walking/Transfer）
4. **Composition API 使用规范** — 全部使用 `<script setup>` 语法
5. **Axios 拦截器封装** — 统一错误处理和 401 重定向

### 问题和改进建议

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| **高德 Key 硬编码** | 🔴 安全 | Key 和 securityJsCode 在多处明文硬编码，应使用环境变量 |
| **密码明文展示** | 🔴 安全 | `UserManagement.vue` 表格直接展示密码列 |
| **数据重复硬编码** | 🟡 维护 | `TripService.vue` 的 metroLines（300+ 行）和 `UserHomePage.vue` 的 lineColorMap 应统一从 API 获取 |
| **重复组件** | 🟡 维护 | `UserHomePage.vue` 和 `AdminHomePage.vue` 代码几乎相同，应抽取为共享组件 |
| **auth.js 未充分使用** | 🟡 架构 | 登录逻辑直接操作 localStorage 而未用 auth.js 工具函数 |
| **Manager.vue 退出登录** | 🟡 Bug | 退出时只清除 `system-user`，未清除 `metro_token`/`metro_role`（User.vue 则正确处理） |
| **ArcGIS 依赖未使用** | 🟢 优化 | package.json 中有 `@arcgis/core`，但代码中未实际使用 |
| **Navigate.vue 组件未使用** | 🟢 清理 | 已定义但未在任何路由中使用 |
| **路由缺少守卫** | 🟡 安全 | 未登录用户可直接访问 `/manager` 或 `/user` 路径 |
| **无 TypeScript** | 🟢 建议 | 虽然是毕设项目，但加入 TS 会加分 |

## 八、后端接口依赖

从代码中可推断后端需提供以下 API：

```
POST   /login                          # 登录
GET    /user                           # 用户列表
POST   /user                           # 新增用户
PUT    /user                           # 更新用户
DELETE /user/:id                       # 删除用户
GET    /api/metro/lines                # 线路列表
GET    /api/metro/stations             # 站点列表
GET    /api/metro/stations/line/:id    # 按线路获取站点
GET    /api/metro/stations/transfer    # 换乘站列表
GET    /api/metro/full                 # 完整地铁数据
GET    /api/metro/detail/:id           # 线路详情
GET    /api/metro/line/:id/schedule    # 线路时刻表（实时轨迹用）
GET    /api/metro/line/:id/trains      # 列车列表（实时轨迹用）
```

## 九、总结

这是一个**完成度约 60%** 的毕设项目。核心的**地图可视化**和**用户交互**功能已实现，UI 设计较用心，但后端管理功能（线路管理/票务/轨迹管理/地图管理）全是占位页面。安全方面有几个需改进的地方（Key 硬编码、密码明文、路由守卫缺失）。总体适合作为毕设项目的起点，剩余功能按现有架构填充即可。
