### 2.3 关键设计决策

| 决策点 | 选型 |
|--------|------|
| 编辑模式 | **模式切换式**（浏览/编辑开关切换），参照 GIS 编辑工具习惯 |
| 线路 CRUD | **折中方案**：表单新增（填名称、颜色等）+ 地图编辑路径 |
| 站点操作 | **完整操作**：拖拽移动 + 双击地图新增 + 右键删除 + 标记换乘 |
| 保存机制 | **手动保存**（脏标记 `_dirty` 追踪 + 批量提交） |
| 地图实现 | 高德 AMap：`PolylineEditor` 编辑路径 + `Marker.draggable` 拖拽站点 |

### 2.4 交互流程

**浏览模式（默认）：**
1. 左侧勾选线路 → 地图实时显示/隐藏对应 Polyline + Marker
2. 点击地图折线 → 选中线路，高亮显示
3. 地图可缩放、平移

**编辑模式（点击开关切换）：**
1. 左侧选中的线路进入可编辑状态
2. 折线显示路径顶点（白色锚点），拖拽修改走向
3. 站点 Marker 可拖拽，`dragend` 事件更新坐标
4. 双击地图空白处 → 弹窗填站名 → 新增站点
5. 右键站点 → 删除/标记换乘
6. 右侧面板表单可精确输入数值

**保存流程：**
- 所有修改对象标注 `_dirty: true`
- 点击「保存全部」→ 收集脏数据 → 批量 API 提交 → 清除脏标记

### 2.5 状态管理

| 状态变量 | 作用 |
|----------|------|
| `isEditing` | 是否处于编辑模式 |
| `hasChanges` | 是否有未保存的修改 |
| `selectedLine` | 当前选中的线路对象 |
| `selectedStation` | 当前选中的站点对象 |
| `line._dirty` | 单条线路是否被修改 |
| `station._dirty` | 单个站点是否被修改 |

### 2.6 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/metro/lines` | 获取所有线路（已有） |
| GET | `/api/metro/stations/line/{lineId}` | 获取线路站点（已有） |
| GET | `/api/metro/transfers` | 获取换乘关系（已有） |
| POST | `/api/metro/lines` | 新增线路 |
| PUT | `/api/metro/lines/{id}` | 更新线路基本信息 |
| PUT | `/api/metro/lines/{id}/path` | 更新线路路径坐标 |
| DELETE | `/api/metro/lines/{id}` | 删除线路 |
| POST | `/api/metro/stations` | 新增站点 |
| PUT | `/api/metro/stations/{id}` | 更新站点信息 |
| PUT | `/api/metro/stations/{id}/position` | 更新站点坐标 |
| DELETE | `/api/metro/stations/{id}` | 删除站点 |
| POST/PUT | `/api/metro/transfers` | 增改换乘关系 |

### 3.3 功能清单

| 功能 | 说明 |
|------|------|
| 线路选择 | 下拉选择要配置的线路，自动加载当前参数 |
| 参数配置 | 首末班时间、发车间隔、站间运行时间、停站时间、列车数量 |
| 预设模板 | 点击后填入对应参数（不自动保存） |
| 时刻表预览 | 根据参数自动计算各列车发车/到达时间 |
| 保存配置 | PUT 到后端，更新 MetroLine 运行字段 |

### 3.4 预设模板参数

| 模板 | 发车间隔 | 站间运行 | 停站时间 | 列车数量 |
|------|---------|---------|---------|---------|
| 高峰期 | 180秒(3分钟) | 120秒(2分钟) | 20秒 | 8-10辆 |
| 平峰期 | 300秒(5分钟) | 180秒(3分钟) | 30秒 | 5-6辆 |
| 夜间模式 | 600秒(10分钟) | 240秒(4分钟) | 40秒 | 3-4辆 |

### 3.5 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|

| PUT | `/api/metro/line/{lineId}/schedule` | 更新时刻表配置 |
